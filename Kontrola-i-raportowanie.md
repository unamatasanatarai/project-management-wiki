Raportowanie, metryki, rejestry, odchylenia, audyty, zgodność i wnioski z realizacji.

## Hasła w tej kategorii

| Hasło | Krótka definicja |
| --- | --- |
| [KPI](Haslo-KPI.md) | KPI (Key Performance Indicator) to mierzalny wskaźnik służący do oceny efektywności procesów, zespołów lub projektów. KPI powinien być konkretny, biznesowo istotny i regularnie monitorowany. Przykłady to velocity, uptime, NPS lub liczba defektów. |
| [OKR](Haslo-OKR.md) | OKR (Objectives and Key Results) to framework do definiowania ambitnych celów oraz mierzalnych rezultatów potwierdzających ich osiągnięcie. Objective opisuje kierunek lub zmianę biznesową, a Key Results konkretne mierniki sukcesu. OKR-y pomagają synchronizować działania zespołów ze strategią organizacji. |
| [Dashboard](Haslo-Dashboard.md) | Wizualny panel prezentujący kluczowe dane projektowe, operacyjne lub biznesowe w czasie rzeczywistym. Dashboard agreguje KPI, statusy, trendy i alerty w jednym miejscu. Ułatwia szybkie podejmowanie decyzji przez management i interesariuszy. |
| [Status Report](Haslo-Status-Report.md) | Regularny raport podsumowujący aktualny status projektu, postępy, ryzyka, problemy, budżet i plan kolejnych działań. Jest podstawowym narzędziem komunikacji z klientem, managementem i sponsorami. Dobry status report powinien być zwięzły, aktualny i oparty na danych. |
| [RAID Log](Haslo-RAID-Log.md) | RAID Log to centralny rejestr Risks, Assumptions, Issues i Dependencies w projekcie. Pomaga PM-owi monitorować czynniki wpływające na sukces projektu i planować działania mitigacyjne. Jest regularnie aktualizowany podczas statusów i governance meetings. |
| [Action / Decision Log](Haslo-Action-Decision-Log.md) | Rejestr decyzji oraz działań ustalonych podczas spotkań projektowych. Zawiera ownerów, terminy realizacji i status wykonania. Zapewnia transparentność, accountability i historię kluczowych decyzji projektowych. |
| [EV / PV / AC](Haslo-EV-PV-AC.md) | EV (Earned Value) to wartość wykonanej pracy, PV (Planned Value) to wartość zaplanowanej pracy, a AC (Actual Cost) to rzeczywisty koszt wykonania. Porównanie tych wskaźników pozwala mierzyć efektywność kosztową i harmonogramową projektu. Są podstawą Earned Value Management. |
| [SPI / CPI](Haslo-SPI-CPI.md) | SPI (Schedule Performance Index) mierzy efektywność realizacji harmonogramu, a CPI (Cost Performance Index) efektywność kosztową projektu. Wartości poniżej 1 oznaczają odpowiednio opóźnienia lub przekroczenie budżetu. Oba wskaźniki są kluczowe w raportowaniu projektowym. |
| [Forecast](Haslo-Forecast.md) | Prognoza przyszłego stanu projektu oparta na aktualnych danych i trendach. Może dotyczyć terminu zakończenia, kosztów, velocity lub ryzyka. Forecasting pozwala wcześniej identyfikować problemy i podejmować działania zapobiegawcze. |
| [Variance Analysis](Haslo-Variance-Analysis.md) | Analiza odchyleń między planem a rzeczywistymi wynikami projektu w zakresie czasu, kosztów lub jakości. Pomaga identyfikować przyczyny problemów i określać działania korygujące. Jest podstawowym elementem project control i governance. |
| [Health Check / Audit](Haslo-Health-Check-Audit.md) | Health Check to przegląd kondycji projektu pod kątem procesów, ryzyk i efektywności delivery, natomiast Audit to formalna kontrola zgodności z procedurami lub regulacjami. Oba działania pomagają wykrywać problemy zanim wpłyną krytycznie na projekt. |
| [RAG Status](Haslo-RAG-Status.md) | RAG (Red, Amber, Green) to system raportowania statusu projektu za pomocą kolorów. Green oznacza brak istotnych zagrożeń, Amber ryzyko wymagające uwagi, a Red krytyczne problemy wpływające na projekt. Pozwala managementowi szybko identyfikować obszary wymagające interwencji. |
| [Compliance](Haslo-Compliance.md) | Zapewnienie zgodności projektu, produktu i procesów z regulacjami prawnymi, standardami branżowymi oraz politykami organizacji. Compliance obejmuje m.in. bezpieczeństwo danych, audyty, licencje i wymagania regulatorów. Brak zgodności może prowadzić do kar finansowych, reputacyjnych i prawnych. |
| [Lessons Learned Register](Haslo-Lessons-Learned-Register.md) | Repozytorium wiedzy zawierające wnioski z sukcesów, błędów i problemów napotkanych podczas projektu. Dokumentuje dobre praktyki oraz obszary wymagające poprawy. Lessons Learned pomagają organizacji zwiększać dojrzałość projektową i unikać powtarzania tych samych błędów. |
