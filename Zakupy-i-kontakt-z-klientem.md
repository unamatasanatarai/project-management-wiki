## Hasła w tej kategorii

| Hasło | Krótka definicja |
| --- | --- |
| [RFI](Haslo-RFI.md) | Request for Information — dokument używany na wczesnym etapie procurementu do zebrania informacji o rynku, dostawcach i dostępnych rozwiązaniach. Nie jest wiążący zakupowo. Pomaga klientowi ocenić możliwości technologiczne, kompetencje vendorów i przygotować shortlistę przed RFP lub RFQ. |
| [RFP](Haslo-RFP.md) | Request for Proposal — formalne zapytanie ofertowe dotyczące rozwiązania konkretnego problemu biznesowego. Dostawcy przedstawiają podejście projektowe, architekturę, harmonogram, skład zespołu, estymacje i koszty. RFP służy do porównania jakości, kompetencji i wartości biznesowej ofert. |
| [RFQ](Haslo-RFQ.md) | Request for Quotation — zapytanie skupione głównie na cenie dla jasno zdefiniowanego produktu lub usługi. Używane, gdy zakres i wymagania są już ustalone, a głównym kryterium wyboru jest koszt, warunki handlowe lub termin dostawy. |
| [ITT](Haslo-ITT.md) | Invitation to Tender — formalne zaproszenie do udziału w przetargu. Zawiera wymagania formalne, kryteria oceny ofert, terminy oraz warunki składania dokumentacji. Najczęściej stosowane w dużych organizacjach i sektorze publicznym. |
| [Tender / Bid](Haslo-Tender-Bid.md) | Oferta przetargowa składana przez dostawcę w odpowiedzi na ITT, RFP lub RFQ. Zawiera zakres prac, harmonogram, koszty, model delivery, ryzyka i warunki handlowe. Wygrany bid często staje się podstawą negocjacji kontraktu. |
| [Proposal](Haslo-Proposal.md) | Dokument sprzedażowo-konsultingowy przedstawiający rekomendowane rozwiązanie dla klienta. Zawiera podejście biznesowe i technologiczne, korzyści, zakres projektu, model współpracy oraz argumenty pokazujące wartość dostawcy. |
| [SoW](Haslo-SoW.md) | Statement of Work — dokument definiujący szczegółowy zakres prac, deliverables, harmonogram, odpowiedzialności, kryteria akceptacji i sposób rozliczenia projektu. SoW ogranicza ryzyko scope creep i stanowi operacyjny załącznik do kontraktu lub MSA. |
| [Contract](Haslo-Contract.md) | Prawnie wiążąca umowa regulująca współpracę między klientem a dostawcą. Określa prawa i obowiązki stron, odpowiedzialność, warunki płatności, własność intelektualną, SLA, kary umowne oraz zasady rozwiązania współpracy. |
| [MSA](Haslo-MSA.md) | Master Service Agreement — ramowa umowa definiująca ogólne warunki współpracy między stronami dla wielu projektów. Standaryzuje kwestie prawne i finansowe, dzięki czemu kolejne inicjatywy uruchamia się poprzez osobne SoW bez ponownych negocjacji całej umowy. |
| [SLA](Haslo-SLA.md) | Service Level Agreement — dokument definiujący mierzalne parametry jakości usług, np. uptime, response time, resolution time czy dostępność supportu. SLA zawiera KPI, poziomy priorytetów incydentów oraz konsekwencje za niedotrzymanie uzgodnionych poziomów usług. |
| [NDA](Haslo-NDA.md) | Non-Disclosure Agreement — umowa poufności zabezpieczająca wymianę informacji biznesowych, technicznych i finansowych między stronami. Chroni własność intelektualną, dane klientów i know-how przed nieautoryzowanym ujawnieniem. |
| [Procurement Plan](Haslo-Procurement-Plan.md) | Plan zakupowy określający strategię pozyskiwania produktów, usług i vendorów w projekcie. Zawiera harmonogram zakupów, budżet, model kontraktacji, kryteria wyboru dostawców oraz zarządzanie ryzykiem procurementowym. |
| [Vendor Evaluation](Haslo-Vendor-Evaluation.md) | Proces oceny dostawców pod kątem kompetencji, jakości usług, doświadczenia, stabilności finansowej, zgodności technologicznej i ryzyka współpracy. Często oparty o scoring, referencje, PoC lub due diligence. |
| [Due Diligence](Haslo-Due-Diligence.md) | Szczegółowa analiza partnera biznesowego przed podpisaniem umowy. Obejmuje weryfikację finansową, prawną, operacyjną, technologiczną i compliance. Celem jest identyfikacja ryzyk związanych ze współpracą lub inwestycją. |
| [Commercials](Haslo-Commercials.md) | Warunki handlowe współpracy obejmujące pricing, model rozliczeń, terminy płatności, rabaty, limity budżetowe i warunki fakturowania. Commercials są kluczowym elementem negocjacji między klientem a dostawcą. |
| [Rate Card](Haslo-Rate-Card.md) | Tabela stawek określająca koszt pracy poszczególnych ról lub kompetencji, np. PM, Developer, QA czy Architect. Najczęściej używana w modelu Time & Materials do estymacji budżetu i wyceny zasobów projektowych. |
| [BAFO](Haslo-BAFO.md) | Best and Final Offer — finalna wersja oferty składana po zakończeniu negocjacji handlowych. Zawiera ostateczne warunki cenowe, zakres i warunki współpracy przed podjęciem decyzji przez klienta. |
| [Call / Discovery Call](Haslo-Call-Discovery-Call.md) | Wstępna rozmowa z klientem mająca na celu poznanie jego potrzeb biznesowych, problemów, oczekiwań i ograniczeń projektowych. Discovery Call pomaga doprecyzować scope, ryzyka oraz przygotować adekwatną ofertę lub estymację. |
| [Pre-sales](Haslo-Pre-sales.md) | Etap wsparcia sprzedaży przed podpisaniem kontraktu, angażujący ekspertów biznesowych i technicznych. Obejmuje discovery, warsztaty, estymacje, przygotowanie ofert, architektury rozwiązania i prezentacji dla klienta. |
| [Client Workshop](Haslo-Client-Workshop.md) | Warsztat roboczy z klientem służący zbieraniu wymagań, definiowaniu scope, mapowaniu procesów lub wypracowaniu koncepcji produktu. Ułatwia alignment interesariuszy i redukuje ryzyko błędnego zrozumienia wymagań. |
